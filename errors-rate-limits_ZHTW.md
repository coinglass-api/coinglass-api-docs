# ❗ 錯誤碼與頻率限制

📡  **API 返回状态码**

CoinGlass API 採用標準的 HTTP 狀態碼來指示請求的成功或失敗。以下表格簡要說明常見的返回碼含義：

| 狀態碼 | 說明                 |
| :-- | :----------------- |
| 0   | 請求成功               |
| 400 | 錯誤的請求（參數錯誤等）       |
| 401 | 未授權（API Key 無效或缺失） |
| 404 | 接口不存在              |
| 405 | 不支援的 HTTP 方法       |
| 408 | 請求逾時（請求處理時間過長）     |
| 422 | 參數格式正確但內容不可接受     |
| 429 | 請求過於頻繁（觸發限流）       |
| 500 | 伺服器內部錯誤            |

🚦 **速率限制（Rate Limits）**

請求頻率限制將依照您所訂閱的付費方案而有所不同。不同等級的方案對應不同的調用頻率上限。

您可前往 [此頁面](https://www.coinglass.com/zh-TW/pricing) 查看詳細說明與限制條款。
